# 図版 指示書（Claude Design 向け）

ministan 本（PHP 静的解析器を一から作るチュートリアル）の**難所**に概念図を足したい。
ここに各図の意図・要素・関係を記す。**正確さより「腑に落ちる」**を優先 —— 数式や網羅は不要。

## 共通スタイル

- **トーン**: やさしく・最小限。教材であり、初学者を脅かさない（本書の軸 `non-rejecting`）。
  既存の雰囲気は Part 0 の ASCII 図（`ソースコード → AST → NodeScopeResolver → Scope → Rule群 → Error群`）。
  これを清書した品位で。
- **言語**: ラベルは日本語＋コード識別子（`Scope`, `int|null`, `mixed` など）。識別子は等幅。
- **配色**: ライトテーマ前提・モノクロ＋差し色1〜2色（型＝青系、エラー/不適合＝赤系、絞り込み＝緑系 など）。
  ダーク両対応が望ましければ CSS 変数で。
- **形式**: **SVG**（テキスト検索・拡大に強い）。幅は本文カラム（〜700px 相当）。各図 1 ファイル。
- **約物**: ダッシュは全角 ` —— `（U+2014・前後スペース）。本文表記に合わせる。
- **命名**: `NN-slug.svg`（例 `02-scope-flow.svg`）。本文からは `![説明](figures/02-scope-flow.svg)` で埋め込む。
- **配置（埋め込み先）**: 各図の節に「**配置**」として記す。本文への挿し込みは著者が
  `![説明](figures/NN-slug.svg)` で行う（デザイナーは図の生成に集中してよい）。

各図に**1行キャプション**（図の主張）を添えてよい。コードや長文の引き写しは避け、構造だけを見せる。

## 図1 —— Part 2: Scope の伝播（`02-scope-flow.svg`）

**主張**: 木を下りながら不変 `Scope` を運び、分岐で枝分かれ・合流する。

**配置**: `02-scope.md` の `NodeScopeResolver`／`processChildren` を説明する節（読み取り・書き込み文脈の区別のあたり）。

**描く内容**
- 小さな AST 木を上から下へ（例: 関数本体の文の並び）。各ノードの脇に、その地点の `Scope`
  （定義済み変数の集合）を吹き出しで示す。
- **代入で変数が増える**: `$a = 1;` を通ると Scope に `$a` が加わる（`{} → {$a}`）。
- **読み取り文脈と書き込み文脈の区別**を視覚化: 代入の左辺＝定義（緑）、右辺や参照＝読み取り（青）。
  読み取りで未定義の変数に当たるとルールが発火（赤）。
- **`if` で枝分かれ**: 条件から then 枝・else 枝の 2 つの Scope に分かれ、`if` の後で
  **合流（楽観的和集合）**して 1 つに戻る。「どちらかで定義されていれば定義済み」を矢印の合流で。

**ラベル例**: `NodeScopeResolver`、`Scope{...}`、`代入＝定義`、`参照＝読み取り`、`合流（和集合）`。

## 図2 —— Part 3: 型の束（`03-type-lattice.svg`）

**主張**: 型は値の集合。`mixed`（最上位）から `never`（最下位）までの包含関係をなす。

**配置**: `03-type-system.md` の「型は『値の集合』」の節の直後。

**描く内容**
- 縦のハッセ図（束）。最上段に **`mixed`**（すべての値）、最下段に **`never`**（空集合）。
- 中段に原子型 `int` `string` `bool` `float` `null`、その下に**定数型** `42`・`'foo'`・`true` を、
  対応する原子型の**部分型**として線で結ぶ（`42` → `int`、`'foo'` → `string`、`true` → `bool`）。
  ＊厳密には `bool = true|false`。`true`/`false` を `bool` の下に結べば正確だが、`bool` を原子型扱いにする簡略化でも可。
- 関係の向きを 1 例だけ注記: **`int ⊇ 42` は Yes、`42 ⊇ int` は Maybe**（非対称）。
- 余白に三値の凡例: `Yes`／`Maybe`／`No`（差し色で）。

**ラベル例**: `mixed（最上位）`、`never（最下位）`、`部分型`、`int ⊇ 42 = Yes / 42 ⊇ int = Maybe`。

## 図3 —— Part 5: Union と絞り込み（`05-narrowing.svg`）

**主張**: 条件が、変数の型を分岐ごとに狭める（narrowing）。合流で union に戻る。

**配置**: `05-narrowing.md` の絞り込みエンジン（`TypeSpecifier`）の節、または冒頭の `int|null` 例の近く。

**描く内容**
- 中央に変数 `$x : int|null`（union を「int と null の2マス」で表現）。
- `if ($x === null)` を分岐点として、**真の枝（truthy）= `$x : null`**、
  **偽の枝（falsy）= `$x : int`** に絞り込まれる様子を、2 本の矢印で。
  `TypeSpecifier` がこの分割を作ることを注記。
- 早期 return を 1 例: 真の枝が `return` で**終わる**と、`if` の後に残るのは falsy 側（`$x : int`）だけ
  —— 「終わる枝は合流に寄与しない」を、合流から外れる点線で。
- else があるときは両枝が **union で合流**して戻る（`A|B`）ことも小さく。

**ラベル例**: `int|null`、`=== null`、`truthy: null`／`falsy: int`、`早期 return → 合流しない`、`mergeWith（union）`。

## 図4 —— Part 8: レベルの正体（`08-levels.svg`）

**主張**: 三値の `Maybe` をレベルで「咎めるか素通りか」決める。厳しさは二段構えで増す。

**配置**: `08-rules-and-levels.md` の「レベルの正体（`RuleLevelHelper`）」の節。

**描く内容**
- 中央に三値の判定: `int` に `mixed` を渡すと **`Maybe`**（たぶん int）。`Yes`→常に OK、`No`→常にエラー。
- 横軸にレベル 0 → max。**閾値**（ministan では 7。「※簡略化した一例」と小さく添える —— 実 PHPStan は
  単一値ではなく、explicit mixed=9 / union=7 / nullable=8 …と段階的）を境に、`Maybe` が左で「素通り（OK）」、
  右で「咎める（エラー）」へ切り替わる様子を、軸上のスイッチで。
- 右下に**二段構え**の注記: レベルを上げるほど (1) **ルールが増え**、(2) **`mixed` にも厳しく**なる。
  （ミニ凡例: level 0=未定義変数等 → 5=引数型 → 6=戻り値型 …）

**ラベル例**: `Yes / Maybe / No`、`int.accepts(mixed) = Maybe`、`閾値`、`低レベル=素通り / 高レベル=咎める`、`ルール増加＋mixed 厳格化`。
